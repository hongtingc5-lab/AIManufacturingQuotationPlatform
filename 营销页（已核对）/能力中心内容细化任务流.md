# 能力中心内容细化任务流

## 目标

对营销页能力中心的子页面进行分批细化，结合现有页面结构、公司工厂图片与 precision 产品图，补充更完整的制造能力、材料、质量、交付与应用场景内容。

页面最终要做到：标题明确、参数可信、流程清楚、图片匹配、CTA 可用。公开页面不得出现内部任务说明、外部参考来源名称或与本品牌无关的公司信息。

## 内容口径原则

- 只输出客户可直接阅读的页面内容，不展示资料来源、整理过程或内部判断说明。
- 参数采用“典型”“可评估”“按图纸要求”“按订单确认”等保守表达，避免绝对承诺。
- 服务、工艺、材料、行业四类入口保持边界清楚，不在同一栏目重复放置不相关内容。
- 每个页面围绕客户关心的问题组织：适用场景、输入资料、工艺/服务范围、质量关注、交付内容与下一步操作。

## 四组推进顺序

每一组先处理对应的内容汇总页，再处理其下属子页面。汇总页负责帮助用户选型和进入详情页，子页面负责承接某一个具体能力的参数、材料、流程、质量与交付说明。

1. 工艺组
   汇总页为 `能力中心—工艺部分.html`，只覆盖制造工艺本身：数控加工、薄板加工、激光切割、水刀切割、注塑成型、聚氨酯铸造、金属 3D 打印、微纳 3D 打印、异形零件加工。医疗零件加工归入行业中心医疗器械场景，材料供应归入材料组。

2. 服务组
   汇总页为 `能力中心—服务部分.html`，覆盖 CNC 加工、模具成型、定制加工、金属冲压、铸造、锻造、3D 打印、快速手板模型等服务入口。

3. 材料组
   汇总页为 `能力中心—材料部分.html`，覆盖铝合金、铜合金、不锈钢、钛合金、工程塑料、复合材料，并补充材料选择逻辑。

4. 表面处理组
   汇总页为 `能力中心—表面处理部分.html`，覆盖阳极氧化、电镀、喷涂、粉末喷涂、钝化、热处理、喷砂、拉丝、抛光等处理方式。

## 单页执行流程

1. 内容整理
   整理该能力的常见定义、可加工材料、典型参数、设计注意事项、质量控制、应用行业和交付流程。

2. 本地适配
   按敏捷智造语气重写，不使用同行公司名、价格承诺或未经确认的认证承诺。参数以“典型”“可评估”“按图纸要求”为主，避免过度绝对化。

3. 页面填充
   保留现有 header、footer、导航和版式，只替换 main 内容区。优先沿用现有 `detail-section`、`info-grid`、`table-card`、`step-list`、`related-grid` 组件。

4. 图片匹配
   优先使用 `precision` 内已处理产品图；工厂环境图用于关于我们、质量保证、能力中心概览或工艺流程模块。图片统一英文命名，避免中文路径进入 HTML。

5. 验证
   检查 HTML 结构、图片路径、锚点、CTA 链接、移动端文字溢出和页面是否存在乱码。

## 图片资产规则

- 新增产品图从中文子文件夹中处理后导出到 `precision` 根目录。
- 文件命名使用小写英文和连字符，例如 `cnc-milled-housing-01.png`。
- 保留原始图片和原始子文件夹，处理图以透明 PNG 输出。
- 自动抠图后进行透明边界裁切，留 16px 左右安全边距。
- 页面图片优先使用产品图，其次使用工厂环境图，避免重复使用同一张图撑满多个页面。

## 本周建议排期

- Day 1：完成任务流、图片资产处理、工艺组 2-3 个重点子页面。
- Day 2：完成工艺组剩余子页面。
- Day 3：完成服务组入口页和相关子内容。
- Day 4：完成材料组与表面处理组。
- Day 5：统一检查导航、图片、文案口径和响应式显示。
