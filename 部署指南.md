# 智能报价系统 - 部署指南

## 当前状态

### 已完成功能 ✅
1. **前端界面** - React + Next.js + Tailwind CSS
2. **文件上传** - 支持拖拽上传STEP/STL文件
3. **3D预览** - Three.js渲染，支持STL完整渲染
4. **AI分析** - 豆包API集成，可进行结构/DFM/报价分析
5. **STEP解析** - 提取几何信息（包围盒、拓扑、曲面类型等）

### 已知限制 ⚠️
1. **STEP文件3D渲染** - 只能显示包围盒，无法渲染完整模型
   - 原因：纯前端JavaScript无法解析STEP中的NURBS曲面
   - 影响：视觉效果受限，但不影响AI分析和报价功能

2. **AI分析速度** - 首次分析可能较慢（30-60秒）
   - 原因：API调用+数据处理
   - 优化：已增加Token限制，应该改善

**原因分析：**
您的STEP文件（`SND-AI-100081-38-J6传动板.step`）包含AP214汽车协议中的B样条曲面，这是工业级CAD格式，需要专业的几何内核才能解析。

## 技术说明

### 为什么纯前端无法完整渲染STEP？

STEP文件是参数化CAD格式，包含：
- **NURBS曲面** - 需要高精度的数学运算
- **B-Rep拓扑** - 面-边-点的复杂关系
- **局部坐标系** - AXIS2_PLACEMENT定义方向
- **裁剪曲线** - 曲面上的裁剪边界

Web端可用的库：
- `occt-import-js` - OpenCASCADE WebAssembly版，但加载慢、内存占用大
- `opencascade.js` - 完整的OCCT编译版，体积大（>50MB）
- 自定义解析器 - 只能提取包围盒，无法渲染曲面

## 替代方案

如果无法添加后端服务，可以：

1. **使用STL文件** - 要求客户提供STL格式（可直接渲染）
2. **手动转换** - 使用FreeCAD桌面版将STEP转为STL
3. **降级方案** - 使用当前的包围盒渲染 + 增强几何信息提取
